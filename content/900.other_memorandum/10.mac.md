## Mac スクリーンショットを撮影するためのショートカット

### 画面全体のスクショを撮影する場合

`Shift` + `command` + `3`

Windowsの `printscreen`

### 範囲を指定してスクショを撮影する場合

`Shift` + `command` + `4`

Windowsの `Shift` + `win` + `s`

### ウインドウを指定してスクショを撮影する場合

`Shift` + `command` + `4` → `Space`

Windowsの `alt` + `printscreen`

## 間違えたパスワードを記憶してしまったときの解消方法

一度を記憶してしまったパスワードなどの認証情報を削除し、再度情報を入力できる状態にする。

### 解消手順

手順は以下の通り。

1. キーチェーンアクセスを表示する。
2. `SourceTree` で認証情報を検索する。
3. 表示された認証情報を削除する。
4. `SourceTree` で再度認証情報を入力する。

### キーチェーンアクセスを表示する

`キーチェーンアクセス` を起動する。 `Command` + `Space` でSpotlight検索を表示し、 `キーチェーンアクセス` を検索して起動する。
`キーチェーンアクセス` では、そのマシンの認証情報をすべて管理している。認証情報を管理しているアプリケーションを検索窓に入力し、該当の認証情報を探す。
例えば`SourceTree` など。

### 表示された認証情報を削除する

認証情報が見つかったら、右クリックして認証情報を削除する。

### 再度認証情報を入力する

ここまでの手順により、マシンから `SourceTree` で入力された認証情報が削除される。
そのため、再度 `SourceTree` からリポジトリへのアクセスなどを試みれば、
認証情報の入力ダイアログが表示され、正しい認証情報を記憶させることができる。

## gradle.propertiesで設定値を管理する

`gradle.properties` を用意し、定数値の感覚で設定すれば、
アプリのバージョンや `compileSdk` / `minSdk` などの値を一つのファイルで一元管理させることができる。

### 設定方法

例えば、以下の `build.gradle` にあるマジックナンバーを定数化し、 `gradle.properties` に外出しして参照する方法を解説する。

```groovy
android {
    namespace = "com.mkt120.sampleapplication"
    compileSdk = 35
    defaultConfig {
        applicationId = "com.mkt120.sampleapplication"
        minSdk = 26
        targetSdk = 35
        versionCode = 100
        versionName = "1.0.0"
        ... 
    }
    ... 
}

```

まず、 `gradle.properties` に外出しする定数を記述する。

```properties
ANDROID_SDK_VERSION=35
ANDROID_MIN_SDK_VERSION=26
ANDROID_VERSION_NAME="1.0.0"
ANDROID_VERSION_CODE=100

```

次に、上で配置した定数を `build.gradle` で参照する。

```groovy
android {
    namespace = "com.mkt120.sampleapplication"
    compileSdk = Integer.parseInt(project.ANDROID_SDK_VERSION)

    defaultConfig {
        applicationId = "com.mkt120.sampleapplication"
        minSdk = Integer.parseInt(project.ANDROID_MIN_SDK_VERSION)
        targetSdk = Integer.parseInt(project.ANDROID_SDK_VERSION)
        versionCode = Integer.parseInt(project.ANDROID_VERSION_CODE)
        versionName = project.ANDROID_VERSION_NAME
        ...        
    }
    ...
}

```

`gradle.properties` に記述した定数は、 `project.XXXXX` の形式で参照することができる。
型を合わせるために `Integer.parseInt()` などでパースするのがポイント。
